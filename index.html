SPC 統計製程管制

Statistical Process Control｜Measure & Control 階段

什麼時候用？
📦 生產過程持續監控⚠️ 早期發現製程異常📉 製程是否穩定🔧 改善前後比較
就像每天量血壓記錄趨勢，SPC 幫製程「量血壓」，超出正常範圍就發出警告。
使用步驟
  • 1每列一筆觀測，同列多值 = 子組
  • 2系統依資料型態自動選出最適合的管制圖
  • 3計算中心線 CL、上界限 UCL、下界限 LCL
  • 4紅點 = 超出界限的異常點，需立即調查
  • 5輸入 LSL/USL 後自動計算 Cp、Cpk
統計公式
① 組平均值 X̄
X̄ = (X₁ + X₂ + … + Xₙ) ÷ n
例：(25.1+25.3)÷2 = 25.2
② 全距 R
R = 最大值 − 最小值
例：25.3−25.1 = 0.2
③ 管制界限
UCL = X̄̄ + A₂×R̄　　LCL = X̄̄ − A₂×R̄
A₂：查表常數（n=2→1.880，n=5→0.577）。白話：中心 ± 3個標準差，超出就是異常。
④ 製程能力 Cpk
Cpk = min[ (USL−X̄̄)÷(3σ), (X̄̄−LSL)÷(3σ) ]
≥1.67 優秀　≥1.33 良好　<1.00 需改善
💡 Nelson 8 條規則：連續 9 點在中心線同側，雖未超出界限也代表製程可能漂移，需注意！

柏拉圖 Pareto Chart

80/20 法則・找出關鍵少數｜Define 階段

什麼時候用？
🎯 找出最重要的問題📋 資源有限時決定優先順序📈 改善前後比較
80% 的問題來自 20% 的原因。柏拉圖幫你找出那 20%，集中火力解決最重要的問題。
使用步驟
  • 1列出所有問題類別與發生次數
  • 2系統自動由大到小排序
  • 3找到累積百分比達 80% 的切點
  • 480% 切點以左 = 關鍵少數，優先解決
統計公式
柏拉圖原則
累積比例 = 前N項總和 ÷ 全部總和 × 100%
將問題由大到小排列，累積到 80% 的項目就是「關鍵少數」，通常只佔所有類別的 20%。
💡 柏拉圖改善後要再畫一次！確認關鍵項目下降，以及是否有新問題浮現。

魚骨圖（石川圖）

6M 根本原因分析｜Define & Analyze 階段

什麼時候用？
🔍 找出問題的根本原因👥 團隊腦力激盪📋 整理複雜的因果關係
魚骨圖像一條魚，魚頭是問題，魚骨是各類原因。透過 6M 分類，系統化找出所有可能的根因。
使用步驟
  • 1清楚定義要分析的問題（魚頭）
  • 26M 分類填入原因：人員/機器/材料/方法/量測/環境
  • 3效應強度 1～5（5=影響最大，靠近魚頭）
  • 4視覺化後聚焦強度高的關鍵原因
統計公式
6M 分類
Man / Machine / Material / Method / Measurement / Mother Nature
每個 M 代表一個可能造成問題的大類。中骨越靠近魚頭，代表效應強度越大。
💡 建議跨部門一起討論，每個人從自己的角度填入原因，找出的根因會更完整！
大骨等長 150px・角度 60°・靠近魚頭 = 效應強度越大（高風險=5 / 中風險=3 / 低風險=1）

直方圖 + 常態檢定

分佈形態分析・Shapiro-Wilk｜Measure 階段

什麼時候用？
📏 了解數據的分佈形狀🔔 確認是否符合常態分佈📐 偏態與峰態分析
直方圖把數據分成區間，顯示每個區間有多少資料，讓你一眼看出數據的「長相」。
使用步驟
  • 1輸入數據（空格或逗號分隔）
  • 2設定組距數（建議 8～15 組）
  • 3看直方圖形狀：鐘形=常態，偏斜=不對稱，雙峰=可能混合兩群
  • 4看常態檢定：p > α = 符合常態，p ≤ α = 不符合常態
統計公式
① 偏態係數
Sk = Σ[(Xᵢ−X̄)³] ÷ (n × σ³)
Sk≈0 對稱　Sk>0 右偏（長尾在右）　Sk<0 左偏（長尾在左）
② 峰態係數
Ku = Σ[(Xᵢ−X̄)⁴] ÷ (n × σ⁴) − 3
Ku≈0 常態峰　Ku>0 尖峰（數據集中）　Ku<0 平峰（數據分散）
③ Shapiro-Wilk 檢定
H₀：數據來自常態分佈
p > 0.05 → 可視為常態；p ≤ 0.05 → 不符合常態，需用非參數方法
💡 很多統計方法（如 t 檢定）假設常態分佈，因此在假設檢定前建議先跑常態性檢定！

DOE 實驗設計

全因子實驗・主效應・交互作用｜Improve 階段

什麼時候用？
🔬 同時測試多個因子的影響💡 找出最佳製程參數⚡ 比一次改一個變數更有效率
傳統「一次改一個變數」效率很低。DOE 讓你同時測試多個因子，用最少實驗次數找出最佳組合。
使用步驟
  • 1設定因子與水準（高/低兩個水準）
  • 2系統產生實驗矩陣（3因子→8次實驗）
  • 3執行實驗並填入結果
  • 4分析主效應：哪個因子影響最大？
  • 5分析交互作用：兩個因子合在一起有特別影響？
統計公式
① 主效應
效應值 = (高水準平均) − (低水準平均)
效應值越大，影響越大。正值=調高有益，負值=調低有益。
② 效應平方和 SS
SS = 效應值² × (總實驗次數 ÷ 4)
SS 越大 = 貢獻率越高，是最重要的因子。
③ 交互作用
AB效應 = (A⁺B⁺均值) − (A⁺B⁻均值) − (A⁻B⁺均值) + (A⁻B⁻均值)
交互作用圖：兩線平行 = 無交互作用；交叉 = 有顯著交互作用，需同時考慮。
💡 DOE 結果告訴你哪些因子最重要，但最佳參數組合還需要反應曲面法（RSM）進一步確認！
⚠️ Plackett-Burman 篩選設計
僅能估計主效應，無法估計交互作用（交互作用與主效應混淆）。
用途：快速從多個因子中篩選顯著因子，再用全因子 DOE 深入分析。

相關性分析 + 散佈圖

Pearson r・線性迴歸・R²｜Analyze 階段

什麼時候用？
🔗 探索兩個變數的關係📐 量化關聯強度📊 建立預測方程式
散佈圖把兩個變數畫在 X-Y 軸，每個點代表一筆數據。點群呈現趨勢代表兩變數有關聯。
使用步驟
  • 1輸入 X（可能原因）和 Y（結果）數值
  • 2看散佈圖：向右上=正相關，向右下=負相關，散亂=無相關
  • 3看 Pearson r：接近±1=強相關，接近0=弱相關
  • 4p < 0.05 = 相關性顯著，不是隨機產生的
統計公式
① Pearson r
r = Σ[(Xᵢ−X̄)(Yᵢ−Ȳ)] ÷ √[Σ(Xᵢ−X̄)² × Σ(Yᵢ−Ȳ)²]
|r|≥0.9 極強　|r|≥0.7 強　|r|≥0.5 中等　|r|<0.3 弱
② 決定係數 R²
R² = r²
R²=0.81 代表 X 解釋了 Y 81% 的變異。R² 越高，解釋力越強。
③ 線性迴歸
Y = b₀ + b₁×X，其中 b₁ = Σ[(Xᵢ−X̄)(Yᵢ−Ȳ)] ÷ Σ(Xᵢ−X̄)²
b₁=斜率，b₀=截距。可用 X 預測 Y 的值。
💡 相關不等於因果！X 和 Y 有強相關，不代表 X 造成 Y。需結合製程知識和實驗設計確認因果。

假設檢定

t-test・F-test・顯著性判斷｜Analyze 階段

什麼時候用？
📊 比較改善前後是否顯著差異🏭 比較兩台機台表現🎯 確認改善效果不是隨機的
假設檢定用「反證法」：先假設沒有差異（H₀），如果數據強烈反對（p < α），就說有顯著差異。
使用步驟
  • 1選擇檢定方法：單樣本t / 雙樣本t / 配對t / F檢定
  • 2設定顯著水準 α（常用 0.05）
  • 3輸入數據，系統計算 t 統計量和 p-value
  • 4p < α → 差異顯著；p ≥ α → 無顯著差異
統計公式
① t 統計量（雙樣本）
t = (X̄₁ − X̄₂) ÷ √(s₁²/n₁ + s₂²/n₂)
t 越大（絕對值），差異越大。自由度由 Welch 公式計算。
② p-value 白話
p = 如果真的沒差異，觀察到這結果的機率
p=0.03 → 只有 3% 機率看到這種差異，很可疑，拒絕 H₀。p < 0.05 視為顯著。
③ F 檢定
F = s₁² ÷ s₂²
比較兩組分散程度。p < 0.05 = 兩組變異數有顯著差異。
💡 統計顯著 ≠ 實務有意義！差異很小但樣本數大時也可能顯著。需結合效果量（Effect Size）判斷。

箱型圖 Box Plot

多組分佈比較・IQR・離群值｜Measure 階段

什麼時候用？
📊 一次比較多組數據分佈🎯 偵測離群值📏 了解中位數和四分位數
箱型圖把數據的中間 50% 畫成箱子，讓你一眼比較多組數據的差異，就像比較不同班級的成績分佈。
使用步驟
  • 1輸入分組數據（格式：組名: 數值1, 數值2, ...）
  • 2看箱子位置：箱子越高 = 中位數越大
  • 3看箱子寬度：越寬（IQR越大）= 數據越分散
  • 4超出鬚線的點 = 離群值，需調查原因
統計公式
① 四分位數
Q1=第25百分位　Q2=中位數　Q3=第75百分位
把數據由小到大排列，Q1是前25%的分界，Q3是前75%的分界。
② IQR 四分位距
IQR = Q3 − Q1
IQR 越大 = 數據越分散。不受離群值影響，比標準差更穩健。
③ 離群值判斷
X < Q1 − 1.5×IQR　或　X > Q3 + 1.5×IQR
超出鬚線的點用圓圈標示。代表不尋常的觀測值，需確認是量測錯誤還是真實異常。
💡 箱型圖只顯示統計摘要，看不到個別數據細節。搭配直方圖一起用效果最好！

目前已知限制：Xbar-S（n>10）、Nelson 部分規則、DOE ANOVA 與 MSA 仍需標準資料集驗證。

定位：本工具為內部工程分析與教育訓練輔助工具，不等同第三方軟體認證或法規驗證系統。

SPC：Xbar-R、I-MR、np chart、Cp/Cpk 等公式依 Montgomery《Introduction to Statistical Quality Control》與常用 SPC 手冊公式設計，已以範例資料做初步驗算。

MSA：Gage R&R 支援 ANOVA 法與平均全距法；判斷門檻採常見 AIAG MSA 判讀：%GRR < 10% 優良、10%~30% 條件接受、>30% 不合格，ndc 建議 ≥ 5。正式使用前仍應以公司核准樣本資料與既有統計軟體交叉確認。
